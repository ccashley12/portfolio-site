Flix

An Angular based movie app

Flix is an Angular-based movie app that provides users with details on various movies, directors, and genres. Users can register, log in, and create a list of their favorite movies. Built with Angular Material and connected to an existing RESTful API.

Project Objective

The goal of Flix is to provide users with an intuitive movie browsing platform that allows them to:

  • Register and log in securely
  • Browse a collection of movies
  • View detailed information about movies, directors, and genres
  • Manage a list of favorite movies
  • Manage their account and edit profile information

Technology Behind Flix

  • Angular - Framework for dynamic web applications
  • Angular Material - UI components for a responsive and accessible design
  • RESTful API - Connects to a backend for real-time movie data
  • JWT Authentication - Secure user login and authentication
  • TypeScript - For code maintainability
  • GitHub Pages - For deployment and hosting

Challenges [&] Growth

Developing Flix was a very big challenge for me. This project helped me gain a deeper understanding of the features in Angular, boosting my ability to develop dynamic web applications. I also explored TypeScript to enhance code maintainability and used TypeDoc to create detailed documentation. As a beginner with Angular, I found the official documentation difficult to navigate, which made the learning process more challenging than anticipated. Nonetheless, overcoming these obstacles improved my adaptability and problem-solving skills, reinforcing my knowledge of these technologies.

Case Study – Flix

Introduction

Project Overview

  • A RESTful API (using Node.js/Express and MongoDB) for managing movie data and user authentication.
  • A Angular-based frontend that lets users browse and save their favorite movies.

Purpose

The primary goal was to master full-stack development by building a complete application from scratch within a guided framework. The project was divided into two phases:

  • Backend (1 month): Developed a secure API with user registration and data management.
  • Frontend (1 month): Created a responsive Angular interface with movies and a favorites feature.

Objective

The objective was to build a robust full-stack solution that demonstrates technical proficiency and clear information presentation. I developed Flix as the sole developer under the guidance of my mentor at CareerFoundry.

Process

Planning

  • Outlined the overall architecture and defined key functionalities by mapping API endpoints and data models.
  • Created wireframes and flowcharts to visualize the user journey—especially for favoriting features.

Backend Development

Constructed a secure RESTful API using Node.js/Express and MongoDB, supporting user registration, login, and data management.

  • User Authentication: Implemented token-based authentication for secure sessions.
  • Data Structuring: Designed scalable data models for movies and users.

Frontend Development

Built this interface with Angular allowing users to browse and save their favorite movies.

  • State Management: Utilized Angular to manage dynamic content and user interactions.
  • Responsive Design: Ensured an optimized UI across desktops and mobile devices.

Testing

  • Conducted regular manual testing and debugging to resolve issues in Angular.
  • Implemented iterative enhancements by fine-tuning API responses, optimizing UI workflows, and improving the favorites functionality.

Conclusion

Final Outcome

Flix as a fully functional movie database that seamlessly integrates a secure RESTful API with a responsive Angular frontend. The application enables users to browse and save their favorite movies.

Challenges [&] Lessons Learned

  • User Authentication [&] Data Structuring: Developing secure, token-based authentication systems and crafting scalable data models significantly enhanced my backend development skills.
  • State Management [&] Responsive Design: Optimizing cross-device interactions and visual consistency emphasized the critical role of iterative QA cycles and responsive design refinement.

Future Directions

  • Database Expansion: Add more movies, series, and categories to offer more data to users.
  • Enhanced Engagement: Introduce features such as user reviews, user followers, and user account themes.
  • Performance Optimization: Optimize API responses and enhance frontend performance to support greater scalability.

Reflecting on these challenges and successes, Flix stands as both a testament to my technical abilities and a commitment to continuous improvement.
