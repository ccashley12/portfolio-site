Pokedex

A simple JavaScript app

This simple JavaScript app is a Pokémon index. You can click each Pokémon listed and a modal will pop up with details and images of each Pokémon. This app fetches data from an API. I have utilized in this project JS, HTML, CSS, Bootstrap, jQuery, and ESLint.

Project Objective

Pokedex is designed to make Pokémon exploration easy and engaging while demonstrating key JavaScript functionalities. The app allows users to:

  • View Pokémon details (name, height, and image) in a Bootstrap-powered modal
  • Sort Pokémon by name or height
  • Search for specific Pokémon by name
  • Swipe through Pokémon within the modal

Technology Behind Pokedex

  • POKÉAPI - Fetches real-time Pokémon data
  • HTML5 [&] CSS3 - Provides structure and styling
  • JAVASCRIPT (ES6) - Handles API requests, DOM manipulation, and event handling
  • BOOTSTRAP 5.3 - Ensures a responsive layout and interactive modals
  • GITHUB PAGES - Hosts the application

Case Study – Pokedex

Introduction

Project Overview

Pokedex is a browser-based JavaScript application that lets users view and explore a list of Pokémon from an external API. As soon as the page loads, users are presented with a dynamic grid of Pokémon. Clicking any Pokémon opens a detailed modal view showing stats, types, and an image.

  • Project Type: Personal / Portfolio
  • Duration: 1 week
  • Tech Stack: HTML, CSS, Vanillla JavaScript, PokéAPI
  • Role: Solo developer - responsible for design, development, and QA
  • Key Features: API integration, modal UI, ESLint formatting, fully brower-compatible

Process & Approach

Planning & Architecture

I structured the app around two main features:

  • A responsive grid that displays Pokémon fetched from the PokéAPI.
  • A modal window that presents detailed info on a selected Pokémon.

This architecture ensured clarity, modularity, and scalability from the start.

API Integration

I loaded the first 150 Pokémon from PokéAPI. When a user clicks on a Pokémon, another fetch request retrieves additional details for the modal view.

Interactive UI & Modal Pattern

  • Modal displays a Pokémon's name, image, type(s), and basic stats.
  • Styled with CSS transitions and responsive design.
  • Accessible: closes via click, ESC key, or keyboard navigation.

Styling & UX

  • Custom CSS Grid for layout and spacing.
  • Buttons styled consistently across browsers.
  • Hover/focus states and animations for better interactivity.
  • Basic CSS reset to ensure cross-browser visual consistency.

Code Quality & Linting

I used ESLint to enforce JavaScript best practices:

  • Clean, modular functions
  • No unused variables or inline styles.
  • Easy to maintain and scale

Browser Compatibility

Tested in:

  • Chrome
  • Firefox
  • Safari
  • Edge

No polyfills needed - all logic written eith ES6 compatibility in mind.

Conclusion

Final Outcome

The final product is a fully responsive and interactive Pokémon browser that delivers a smooth, user-friendly experience across all modern devices. It features a clean, accessible modal for viewing detailed Pokémon information, ensuring both functionality and ease of use. Built entirely with vanilla JavaScript, the app is lightweight, fast, and free of external frameworks.

Challenges [&] Growth

Through this project, I learned how to architect a clean and accessible modal UI using JavaScript and CSS, with attention to both functionality and user experience. I gained a deeper understanding of writing modular, reusable functions to keep the codebase organized and maintainable. Following ESLint rules from the start helped me write cleaner, more consistent code, and I saw firsthand how small details—like loading indicators and keyboard support—can significantly enhance the overall user experience.

Future Directions

  • Add search and filtering functionality
  • Implement lazy loading for images
  • Explore TypeScript integration for stronger type safety
